Robot Sweatshop

Lightweight, unopinionated Continuous Integration

Download this project as a .zip file Download this project as a tar.gz file

Jenkins is horrible to maintain and is problematic when automating installation and configuration. Drone assumes that you use Docker. Travis-CI is difficult to self-host. All of these frameworks are highly opinionated in one way or another, forcing you to do things their way.

Robot Sweatshop is a single-purpose CI server that runs collections of arbitrary scripts when it needs to, usually when new code is pushed. There's no assumptions about what you want to report, what front-end you need, or even what repositories you want to clone because you can do that better than I can. It's just you, your code, and the scripts that test and deploy it.

For more information, visit the GitHub repository

TL;DR

  1. install ZMQ as described in the EZMQ gem
  2. gem install robot_sweatshop
  3. sweatshop start
  4. sweatshop job example --auto
  5. curl -d '' http://localhost:8080/payload-for/example
  6. cat .robot_sweatshop/log/job-worker.log
